# Week 6 LLM Usage Report

During Week 6, AI/LLM tools were used as support tools for coding, documentation, Sprint Review preparation, customer-handover content, CI verification, and transcript handling. The team reviewed and corrected all outputs before incorporating them into the project.

## Models and Tools Used

- Codex 5.5-5.4 Medium
- ChatGPT 5.5 High
- Google AI Studio
- Gemini
- MiMo V2.5

## How AI/LLM Tools Were Used

- Assisted with writing and structuring the customer-handover document, including non-technical usage instructions, deployment guidance, configuration and secrets handling, and troubleshooting sections.
- Helped draft and refine Sprint 4 and Sprint 5 planning documentation, including milestone goals, PBI acceptance criteria, and Story Point estimates.
- Reviewed Kotlin and Python code changes for merge-readiness, suggested fixes for edge cases, and checked backend endpoint correctness.
- Assisted with AGENTS.md and CONTRIBUTING.md creation, ensuring consistent structure and alignment with the maintained documentation set.
- Helped structure the Sprint Review transcript from meeting audio and sanitized it for public publication.
- Assisted with README.md updates to ensure the public repository entry point met Assignment 6 requirements for a first-time reader.
- Helped verify CI workflow configurations and quality-requirement-test alignment.
- Assisted with grammar and structure review of week6 report files.
- Helped convert design and layout suggestions into Android XML layout adjustments for responsive statistics UI.

## Human Review and Verification

- Team members manually reviewed all AI/LLM-generated content before committing it to the repository.
- Code suggestions were tested against the repository structure, backend endpoints, and expected Android behaviour.
- Documentation was cross-checked against the actual product state, deployment configuration, and customer meeting outcomes.
- The Sprint Review transcript was reviewed and sanitized by the team to remove participant-identifying details before publication.
- All Sprint 4 issue-linked PRs were reviewed by a different team member than the author before merge, consistent with the Definition of Done.

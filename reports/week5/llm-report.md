# Week 5 LLM Usage Report

During Week 5, AI/LLM tools were used as support tools for coding, review, documentation, design, CI, and transcript preparation. The team reviewed and corrected the outputs before using them in the project.

## Models and Tools Used

- Codex 5.5
- Codex 5.5 Medium
- Codex 5.4
- Chat GPT 5.5 High
- Google AI Studio
- Gemini
- mimo V2.5

## How AI/LLM Tools Were Used

- Checked code for merge conflicts.
- Reviewed teammate pull requests and suggested fixes when issues were found.
- Helped finalize the app interface design, including the manual record entry design in History.
- Helped convert design images into Android XML layout files.
- Searched for critical bugs in Kotlin code and helped make Kotlin code cleaner and more structured.
- Helped with backend endpoints, tests, and checking individual code changes.
- Clarified expected behavior of framework features used in the project.
- Helped write and structure quality requirements and documentation.
- Helped with CI configuration.
- Helped with grammar checks in documentation.
- Helped convert Sprint Review audio into a text transcript.

## Human Review and Verification

- Team members manually reviewed AI/LLM suggestions before applying them.
- Code suggestions were checked against the repository structure and expected behavior.
- Backend and test-related work was checked with tests, CI evidence, or manual review where applicable.
- Documentation was compared with repository evidence and Sprint Review materials.
- The public transcript and reports were edited and sanitized by the team before publication.

## What Was Not Delegated to AI/LLM Tools

- Final product decisions and Sprint priorities.
- Customer approval and Sprint Review decisions.
- Pull request review responsibility and merge decisions.
- Work with private credentials, private recording links, or customer-identifying details.
- Automatic acceptance of generated code or text without human review.

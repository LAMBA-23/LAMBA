# Week 4 LLM Usage Report

AI/LLM tools were used during Week 4 as engineering assistants for backend implementation, automated testing, quality documentation, CI troubleshooting, frontend development, and code review response.

## Tools Used

- Gemini
- AI Google Studio
- Codex 5.5 Medium / Codex 26.616.6631.0
- mimo V2.5

## How AI/LLM Tools Were Used

### Frontend

- Gemini was used as a reference tool when converting design images into Android .xml layout files.
- AI support was used to assist in searching for critical bugs in Kotlin code and structuring Kotlin code.
- AI support was used to check grammar in documentation files.

### Backend

- Codex 26.616.6631.0 was used as a reference tool to clarify FastAPI and PostgreSQL behavior, review code fragments, suggest readability improvements, and identify possible merge conflicts.
- mimo V2.5 was used as an assistant when implementing backend endpoints, writing unit tests, integration tests, and quality requirement tests.

### Audio and Transcript Support

- AI Google Studio was used as a tool to assist in converting audio into text for documentation and reporting work.

### Code Review and Merge Support

- Codex 5.5 Medium / Codex 26.616.6631.0 was used to assist in checking code for merge conflicts and reviewing a teammate's pull request with fix recommendations.
- mimo V2.5 was used to assist in resolving merge conflicts and fixing code and documentation comments.

## Outcome

LLM tools were used as supporting engineering tools rather than as a replacement for implementation or review responsibility. The team made all design, implementation, and review decisions, using AI assistance to speed up code understanding, test creation, documentation, CI troubleshooting, frontend development, and iterative fix cycles during Week 4 work.

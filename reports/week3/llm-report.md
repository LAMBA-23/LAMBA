# Week 3 LLM Usage Report

AI/LLM tools were used during Week 3 for code review, implementation support, code cleanup, and documentation-related work.

## Tools Used

- Gemini
- AI Google Studio
- Codex 5.5 Medium / Codex 26.616.6631.0
- mimo V2.5

## How AI/LLM Tools Were Used

### Frontend

- Gemini was used to help convert design images into Android `.xml` layout files.
- AI support was used to search for critical bugs in Kotlin code.
- AI support was also used to make Kotlin code more structured and clean.

### Backend

- AI support was used for short reviews to better understand the existing backend code.
- AI tools were used to search for critical bugs in Python code.
- `mimo V2.5` was used during work on backend endpoints and tests.

### Audio and Transcript Support

- AI Google Studio was used to help convert audio into text for documentation and reporting work.

### Code Review and Merge Support

- `mimo V2.5` was used to review teammates' pull requests, resolve merge conflicts, and fix code and documentation comments.
- `Codex 26.616.6631.0` was used to check code for merge conflicts.
- `Codex 26.616.6631.0` was also used to review a teammate's pull request and provide recommendations for fixes where needed.

## Outcome

LLM tools were used as supporting engineering tools rather than as a replacement for implementation or review responsibility. They helped speed up code understanding, review, cleanup, and issue detection during Week 3 work.
